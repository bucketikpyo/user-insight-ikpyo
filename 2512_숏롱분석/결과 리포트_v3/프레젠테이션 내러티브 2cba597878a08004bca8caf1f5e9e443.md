# 프레젠테이션 내러티브

## 프레젠테이션 흐름

### 1. 오프닝: 문제 제기 (Hook)

**"우리가 밀고 있는 상품이 정말 좋은 상품일까요?"**

- PV 상위 10% 상품이 전체 조회의 88%를 차지
- 우리는 이 상품들이 "잘 팔리니까 좋은 상품"이라고 가정해왔음
- 근데 정말 그런가? → **검증해봤습니다**

---

### 2. 핵심 발견 (1문장으로 먼저)

> "PV가 높다고 매력적인 상품이 아니었습니다. 상관관계가 없었어요."
> 
- r = 0.077 (거의 0)
- 이게 뭘 의미하냐면...

---

### 3. 그래서 뭐가 문제인가? (So What)

**2x2 매트릭스 보여주기**

```
         PV 낮음          PV 높음
       ┌─────────────┬─────────────┐
매력    │   C1        │    C3       │
높음    │ 숨겨진 보석    │  스타 상품    │
       │   (9개)     │   (11개)     │
       ├─────────────┼─────────────┤
매력    │   C4        │    C2       │
낮음    │  저성과       │  문제 상품    │
       │  (14개)     │   (6개)      │
       └─────────────┴─────────────┘

```

**강조 포인트:**

- **C1 (숨겨진 보석)**: 매력적인데 아무도 안 봄 → **기회 손실**
- **C2 (문제 상품)**: 별로인데 많이 봄 → **왜?**

---

### 4. 왜 이런 일이 생기나? (원인 분석)

**C3 (스타 상품)은 어떻게 성공했나?**

```
[스타일링샷 축적] → [콘텐츠 노출] → [PV 상승] → [구매/리뷰] → [추가 SS]
        ↑                                              │
        └──────────────── 선순환 ────────────────────────┘

```

**C1 (숨겨진 보석)은 왜 실패했나?**

- 초반에 스타일링샷이 안 쌓임
- 선순환 진입 자체를 못 함
- 매력은 있는데 **시작점이 없었던 것**

---

### 5. 근거 제시 (데이터로 설득)

**스타일링샷 → PV 효과 (코호트별)**

| 코호트 | SS 효과 (회귀계수) | 패턴 |
| --- | --- | --- |
| C3 (스타) | 11.28 | SS 먼저 → PV 나중 ✅ |
| C2 (문제) | 56.41 | PV 먼저 → SS 나중 |
| C1 (숨겨진) | 1.32 | 둘 다 없음 |

**핵심 인사이트:**

- C3는 SS가 PV를 만들었음 (인과관계)
- C1은 SS 자체가 없어서 측정 불가 상태

---

### 6. 그래서 뭘 해야 하나? (제언)

**가설:**

> "C1에 초반 스타일링샷을 인위적으로 추가하면, C3처럼 선순환에 진입할 수 있다"
> 

**실험 제안:**

| 항목 | 내용 |
| --- | --- |
| 대상 | 숨겨진 보석 9개 상품 |
| 방법 | 콘텐츠팀과 협업해서 SS 부스팅 |
| 측정 | PV 변화, 선순환 진입 여부 |
| 기간 | N주 |

---

### 7. 클로징: 기대 효과 & 한계

**기대 효과:**

- 숨겨진 좋은 상품 발굴 → 고객 만족 ↑
- 상품 다양성 증가 → 플랫폼 건강성 ↑
- (성공 시) 신상품 런칭 전략에 적용 가능

**한계 & 다음 스텝:**

- 소파 80개 한정 → 카테고리 확장 필요
- SS 몇 개가 임계점인지 추가 분석 필요
- 매력도 → 실제 구매 전환 연결 검증 필요

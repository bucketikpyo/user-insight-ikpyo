# 테크블로그 재구성 변경사항

## 목표
리서치 리포트 스타일 → 인사이드아웃 글처럼 협업 여정과 스토리텔링 중심으로 재구성

## 주요 구조 변경

### Before (리포트 스타일)
```
1. 도입부
2. UX Researcher와 Data Analyst의 만남
3. 첫 번째 발견: PV와 매력도는 무관하다
4. 두 번째 발견: 스타일링샷이 PV를 만든다
5. 세 번째 발견: 카테고리마다 다르다
6. 실무에 적용하기: 액션 아이템
7. 한계와 의의
8. UXR 🤝 DA: 협업이 만든 시너지
```

### After (스토리텔링 스타일)
```
1. 보물 찾기 게임의 시작 (유지)
2. 처음엔 숫자에서 출발했습니다 ⭐ NEW
   - DA의 로그 데이터 분석
   - 숫자만으로는 알 수 없었던 "왜"
   - 전환점: 고객에게 직접 물어보기로 결정
   - 리서치 설계 과정
3. 충격적인 발견: PV와 매력도는 무관하다
4. 더 깊이 파고들다 ⭐ 통합
   - 발견2, 3을 하나로 통합
   - 기술 디테일 최소화
   - 인사이트 중심으로 재구성
5. 전사적으로 활용되는 인사이트 ⭐ 중간 배치
   - 임팩트를 중간에 배치 (인사이드아웃 스타일)
   - 다양한 팀 활용 사례 강조
6. 한계와 의의
7. UXR 🤝 DA: 협업이 만든 시너지 ⭐ 재구성
   - 협업 여정 요약
   - "숫자→고객 목소리" 전환점 강조
```

## 세부 변경 사항

### 1. Section 2: "처음엔 숫자에서 출발했습니다" (신규)
**변경 내용:**
- 기존 "UX Researcher와 Data Analyst의 만남" 섹션을 재구성
- DA가 먼저 분석을 시작한 과정 강조
- "숫자만으로는 알 수 없었다"는 고민 부각
- "전환점: 고객에게 직접 물어보기로 했습니다" 서브섹션 추가
- 인사이드아웃 글처럼 "다시 고객에게 초점" 전환점 명확히

**핵심 메시지:**
> 로그 데이터만으로는 '무엇이 일어났는지(What)'는 보여주지만, '왜 그런지(Why)'는 말해주지 않았다.

### 2. Section 4: "더 깊이 파고들다" (통합)
**변경 내용:**
- 기존 "두 번째 발견", "세 번째 발견"을 하나로 통합
- 제목을 질문 형식으로 변경: "왜 어떤 상품은 발견되고, 어떤 상품은 묻힐까?"
- Lag Correlation, Granger Causality 등 기술 용어를 각주로 간소화
- 인과관계 검증 과정을 스토리로 풀어냄

**Before:**
```
## 📊 두 번째 발견: "스타일링샷이 PV를 만든다"
### 인과관계 검증: 정말 SS가 PV를 만드는가?
**Lag Correlation 분석***
**Granger Causality 검증***
```

**After:**
```
## 🔍 더 깊이 파고들다: 왜 어떤 상품은 발견되고, 어떤 상품은 묻힐까?
### 시간의 흐름을 따라가다
### 스타일링샷이 열쇠였다
*시계열 분석 기법을 활용해 인과관계를 검증한 결과...
```

### 3. Section 5: "전사적으로 활용되는 인사이트" (중간 배치)
**변경 내용:**
- 기존 "실무에 적용하기" 섹션을 Section 5로 이동 (기존 Section 6에서)
- 인사이드아웃 글처럼 임팩트를 중간에 배치
- "다양한 팀에서 활용 중입니다" 서브섹션 추가
- 광고팀, MD팀, 추천팀, 프로덕트팀 활용 사례 명시

**핵심 메시지:**
> 분석만으로는 의미가 없습니다. 이번 리서치의 진짜 가치는 **실제로 비즈니스에 적용**되고 있다는 점입니다.

### 4. Section 7: "UXR 🤝 DA: 협업이 만든 시너지" (재구성)
**변경 내용:**
- 협업 여정을 시간 순서로 재정리
- "처음에는 숫자에서 출발 → 고객에게 물어보기로 결정" 전환점 강조
- What-Why-How 프레임워크로 협업 가치 정리
- 마무리를 더 따뜻하고 미래지향적으로 변경

**Before:**
```
처음 이 프로젝트를 시작할 때, 저희는 로그 데이터만으로 답을 찾으려 했습니다.
```

**After:**
```
이번 프로젝트를 돌아보면, 저희의 여정은 하나의 전환점을 중심으로 나뉩니다.

**처음에는 숫자에서 출발했습니다.** Data Analyst가 로그 데이터를 분석하며...
**그래서 고객에게 직접 물어보기로 했습니다.** UX Researcher와 협업하여...
```

## 톤 변경 사항

### 1. 리포트 톤 → 스토리텔링 톤
**Before:**
- "첫 번째 발견", "두 번째 발견", "세 번째 발견"
- 기술 용어 전면 배치
- 결과 나열 중심

**After:**
- "충격적인 발견", "더 깊이 파고들다"
- 기술 용어를 각주/보조 설명으로
- 과정과 고민 중심

### 2. 기술 디테일 최소화
**Before:**
```
**Lag Correlation 분석***
*Lag Correlation: 시간차를 두고 두 변수 간의 상관관계를 분석하는 방법*
- **C3(스타 상품)**: SS가 일정 기간 후 PV에 지속적 영향
- **C2(과대노출)**: SS가 단기간 내 PV에 강한 영향
```

**After:**
```
시계열 분석 기법*을 활용해 인과관계를 검증한 결과, 명확한 결론을 얻을 수 있었습니다.
*Lag Correlation(시간차 상관분석)과 Granger Causality(인과관계 검증) 기법을 활용했습니다.
```

### 3. 문장 톤 자연스럽게 변경
**Before:**
- "제한된 샘플로 분석하여 전체 상품군으로 일반화 어려움"
- "SS 2~5개 추가라는 구체적 액션"

**After:**
- "제한된 샘플로 분석하여 전체 상품군으로 일반화 어려움" (유지하되 자연스럽게)
- "스타일링샷 추가라는 구체적이고 실행 가능한 액션을 찾았습니다."

## 인사이드아웃 글과의 유사점

### 1. 협업 여정 강조
- ✅ "숫자에서 출발 → 한계 인식 → 고객 목소리로 전환" 구조
- ✅ 전환점을 명확히 부각

### 2. 임팩트 중간 배치
- ✅ 전사적 활용 사례를 중간(Section 5)에 배치
- ✅ 다양한 팀의 활용 사례 구체적으로 명시

### 3. 스토리텔링 톤
- ✅ 리포트 느낌 제거
- ✅ 고민과 과정 중심
- ✅ 기술 용어 최소화

### 4. 마무리 메시지
- ✅ 협업의 가치 강조
- ✅ 미래지향적 메시지
- ✅ 고객 중심 문화 강조

## 기대 효과

1. **독자 몰입도 향상**: 리포트 → 여정 스토리
2. **협업 가치 부각**: UXR과 DA 협업의 전환점과 시너지 강조
3. **임팩트 강조**: 비즈니스 적용 사례를 중간에 배치하여 실용성 부각
4. **톤 일관성**: 오늘의집 테크블로그(인사이드아웃 등)와 톤 일치
5. **기술 접근성**: 전문 용어를 최소화하여 다양한 독자층 고려
